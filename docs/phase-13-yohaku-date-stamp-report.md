# Phase 13 Yohaku Date Stamp Report

## 目的

「残心で何が残るのか」を明確にするため、心の余白・日付印UIを追加した。

---

## 1. 改善前の課題

- 「何が残るのか」がまだ分かりづらかった
- 日付が管理情報として見えやすかった（「残した日: 今日」のような表示）
- メモカードが普通のメモカードに見えやすかった
- エディタに「今日の余白」感が足りなかった
- 保存ステータスが少し長く詩的すぎた（「余韻を保存しました」など）

---

## 2. 追加した概念

残心で残るもの:

1. **その日の言葉**
2. **その日の余白**
3. **その日付のしるし**

日付は単なる管理情報ではなく、紙に残ったしるしとして扱う。

---

## 3. 日付印UI

### 実装内容

- `src/lib/zanshinDate.ts` を新規作成
  - `formatZanshinDate()`: ISO文字列を `YYYY.MM.DD` 形式に変換
  - `getYohakuLabel()`: 時間帯から「朝/昼/夕/夜の余白」を生成
  - `getZanshinStamp()`: 日付とラベルをまとめて返す
- `src/components/ZanshinDateStamp.tsx` を新規作成
  - 日付印を表示する専用コンポーネント
  - `compact` プロパティで横並び/縦並びを切り替え

### 採用した表示

```txt
夜の余白
2026.05.19
```

または、コンパクト時:

```txt
夜の余白　2026.05.19
```

### デザイン方針

- 日付は点（`.`）区切り — スラッシュより静かで紙の印に合う
- 余白ラベルは明朝、日付は sans-serif
- 色は控えめ: ラベル `rgba(31,27,24,0.68)` / 日付 `rgba(31,27,24,0.46)`
- フォントサイズは 0.72〜0.76rem と小さく、主役にしない

---

## 4. 時間帯ラベル

### 実装したラベル

- **朝の余白**: 5:00 - 10:59
- **昼の余白**: 11:00 - 15:59
- **夕の余白**: 16:00 - 18:59
- **夜の余白**: 19:00 - 4:59

### 基準

- `createdAt` を基準に判定
- そのメモが「置かれた時」の余白として扱う
- データには保存せず、表示時に派生生成する

## 6. エディタの改善

### 変更内容

- エディタ上部（紙面の上）に日付印を縦並びで表示
- 「残した日: 今日」の表示を削除
- 保存ステータスを簡潔に変更
  - 「余韻を保存しました」→「保存済み」
  - 「余韻を静かに整えています」→「保存中」

### レイアウト

```
← 戻る　　　　　　　　[favorite] [delete]

        夜の余白
       2026.05.19

┌─────────────────────────┐
│  [紙面: タイトル・本文]  │
└─────────────────────────┘

[保存中...]
```

- `saved`: "保存済み" / "Saved"

## 11. 残っている課題

### データ構造の拡張性

- 現在は `createdAt` を基準に日付印を表示している
- 将来的に「編集時の余白」も見せたい場合は `updatedAt` も活用できる
- ただし MVP では createdAt 基準のままで十分

### 時間帯ラベルのローカライゼーション

- 現在は日本語のみ（「朝/昼/夕/夜の余白」）
- 英語では "Morning space" / "Afternoon space" などが考えられる
- ただし MVP では日本語中心で問題なし

### 保存アニメーション

- Phase 13 では保存ステータスを控えめに変更した
- 次の Phase 14 で、保存後に日付印が紙に定着するアニメーションを実装予定

---

## 12. 次フェーズへの引き継ぎ

次の **Phase 14** では、保存後に日付印が紙に控えめに定着するアニメーションを実装する。

### Phase 14 の方向性

- 保存完了時に、日付印が和紙に滲むように静かに定着する
- 「残心」の世界観を損なわない、ごく控えめなアニメーション
- 派手な閃光やパーティクルではなく、墨が紙に染み込む感覚
- アニメーション時間は 600〜800ms 程度を想定

### 引き継ぎ事項

- 日付印の DOM 構造とクラス名は Phase 13 で確定
- CSS アニメーションは Phase 14 で追加
- 保存ステータス（「保存中」→「保存済み」）の表示は維持

---

## まとめ

Phase 13 では、「残心で何が残るのか」を明確にするため、日付印UIを追加した。

- 日付は管理情報ではなく、**紙に残ったしるし**として扱う
- 時間帯ラベル（朝/昼/夕/夜の余白）で、その言葉が置かれた時の空気感を伝える
- メモカード・エディタに日付印を配置し、「心の余白に言葉が残る」体験を強化
- 保存ステータスを控えめに変更し、次の Phase 14 のアニメーション実装に備える

すべての機能が正常に動作し、ビルドも成功している。
