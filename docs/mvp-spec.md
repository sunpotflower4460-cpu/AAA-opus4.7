# 残心 — MVP仕様 / MVP Specification

## MVPの目的

「残心」の最小体験を成立させること。

```
起動する
　↓
書く
　↓
静かに保存される
　↓
読み返せる
　↓
探せる
　↓
大切なメモを残せる
```

ここまでを作る。それ以上は、まだ作らない。

---

## 必須画面

### 1. Notes List — メモ一覧

**表示要素：**

| 要素 | 詳細 |
|------|------|
| アプリ名 | 「残心」 |
| サブタイトル | 「Zanshin Notes」 |
| 検索バー | タイトル・本文を横断検索 |
| メモカード一覧 | 更新日時順（新しい順） |
| 新規作成ボタン | FAB（右下固定、55px） |
| 空状態メッセージ | 以下参照 |

**空状態メッセージ：**

```
まだ、言葉は置かれていません。
No words have settled yet.
```

---

### 2. Editor — メモ編集

**表示要素：**

| 要素 | 詳細 |
|------|------|
| 戻るボタン | 左上 |
| タイトル入力 | プレースホルダー：「題、あるいは無題。」 |
| 本文入力 | プレースホルダー：「ここに言葉を置いてください。」 |
| 自動保存ステータス | 以下参照 |
| お気に入りボタン | トグル（金色） |
| 削除ボタン | 朱色、確認ダイアログあり |

**自動保存の文言例：**

```
余韻を保存しました
Saved in stillness
```

**保存ステータスの表示方針：**
- 派手にしない
- テキストまたは微細なアイコンのフェードで表現
- 完了後は静かに消える（残りすぎない）

---

### 3. Read Mode — 読み返しモード

MVPではEditorと兼用でよい。  
将来的には読み返し専用の静かな表示モード（フォントを大きく、UIを隠す）を追加できるように、構造だけ意識しておく。

```typescript
export type Note = {

  title: string;
  body: string;
  createdAt: string;   // ISO 8601
  updatedAt: string;   // ISO 8601
  isFavorite: boolean;
  locale?: "ja" | "en";
};
```

## 保存方針

MVPではローカル保存。

**保存キー：**

```
zanshin.notes.v1
```

- `localStorage` で開始してよい
- 将来的に `IndexedDB` へ移行できるように、保存処理は `lib/storage.ts` などに分離する
- 保存処理を直接コンポーネントに書かない

---

## MVPでやらないこと

以下の機能はMVPには含めない。Phase 3完了後に改めて検討する。

| 機能 | 理由 |
|------|------|
| ログイン | MVPの範囲外 |
| クラウド同期 | MVPの範囲外 |
| AI機能 | 静けさを損なう可能性がある |
| 課金 | MVP完成後に検討 |
| Markdown完全対応 | 本文入力のシンプルさを優先 |
| 複雑なタグ管理 | 検索で代替 |
| 共同編集 | 個人の余白を守るため不要 |
| App Store申請 | Phase 8以降 |
| Cloudflareへの途中デプロイ | Phase 3 MVP完成後のみ |

---

## 検索仕様

- タイトル・本文を対象にリアルタイム検索
- 大文字小文字を区別しない
- ひらがな・カタカナの変換は将来検討（MVPでは厳密一致）

---

## お気に入り仕様

- 一覧画面でお気に入りのみフィルタできる（任意）
- お気に入りのメモにはゴールドのマーカーを表示
- トグル操作で即時反映・自動保存

---

## 削除仕様

- 削除前に確認ダイアログを表示
- 確認文言例：
  ```
  この言葉を手放しますか？
  Release this note?
  ```
- 削除は取り消し不可（MVPでは Undo なし）
