# Phase 9 Typography / Texture / Writing Experience Report

## 目的

テキスト量を減らし、余白・質感・フォント・書字体験を強化することで、
「書いたあとに心が残るメモ帳」としての体験を高めた。

このフェーズの合言葉：

> 説明を削り、紙を深め、文字に呼吸を与える。

機能追加は一切行わず、UI/UX の質感だけを磨き込んだ。

---

## 改善前の課題

スクリーン上で以下が課題として見えていた。

- ホームカード内の「残心とは」の説明が常時大きく出ていて、
  「説明感」が「余韻」より前に出ていた。
- 英語コピー（タグライン、サブタイトル、空状態英文）が letter-spacing 強めで、
  画面の密度を上げていた。
- メモを書く時の本文フォント・行間・紙面感がまだ弱く、
  「ただのテキスト入力欄」に近かった。
- ノートカード内の `最後の余韻` / `残した日` ラベルがやや強く、
  カード全体が詰まって見えた。
- ホーム画面の `+` ボタン（FAB）が真っ黒で、画面の静けさに対してやや強かった。

---

## 修正した内容

### テキスト整理

- ホームカードのタグラインを短く差し替え。
  - 旧：`書いたあとにも、心がそこに残るメモ帳。`
  - 新：`書いたあとにも、心が残る。`
- ホームカード下部の `残心とは、書き終えたあとにも 心が静かに残っていること。`
  という説明は **ノートが 1 件もないときだけ** 表示するように変更。
  ノートが置かれたら静かに引っ込み、書く場所に席を譲る。
- ホームカード内の英語タグライン（`A quiet place for words that remain.`）は
  常時表示をやめ、必要な箇所（空状態の英文サブコピー）に役割を集約。
- 空状態の本文を、説明調から余韻調に書き換え。
  - 旧：`残心とは、書き終えたあとにも 心が静かに残っていること。`
  - 新：`書き終えたあとに残る、心のしずけさをここに。`
- 空状態の英語サブコピーも、機能説明から世界観に揃え直し。
  - 旧：`The first quiet trace begins here.`
  - 新：`A quiet place for words that remain.`
- エディタのタイトル下にあった `A quiet place for words that remain.` の
  英語コピーは削除。書く前に英文を読ませない方針に変更。

### フォント改善

- CSS 変数で 3 つの役割を分離。
  - `--font-display`：見出し・タイトル用、静かな明朝
  - `--font-writing`：本文入力用、明朝（読みやすさを優先）
  - `--font-ui`：UI 操作・補助文用、sans
- `body` の `font-family` を `var(--font-ui)` に統一。
- `.font-mincho` を `var(--font-display)` に紐付け。
- 新しい `.font-writing` クラスを追加し、書字フォントを再利用可能に。
- 外部 Web フォントは **読み込まない**。MVP 段階では iOS / macOS 標準の
  Hiragino Mincho / Yu Mincho / Hiragino Sans を中心とし、
  Noto Serif JP / Shippori Mincho などはフォールバックとしてのみ宣言。
  （ライセンス・読み込み速度・App 化を考慮）

### エディタ改善

- エディタ本体を `.editor-paper` で包み、ただの入力欄ではなく
  「一枚の和紙」として扱う：
  - 背景：`rgba(251, 248, 241, 0.72)`
  - 枠線：`rgba(31, 27, 24, 0.08)`（薄い和紙の縁）
  - 角丸：`34px`（黄金比スケール）
  - 影：外側に柔らかい長い影、内側に 1px の白いハイライト
- 本文 `<textarea>` に `.writing-textarea` を適用：
  - `font-size: 18px`（旧 16px）
  - `line-height: 2.05`（旧 1.85）
  - `letter-spacing: 0.03em`
  - 色：`rgba(31, 27, 24, 0.88)`
  - プレースホルダー：`rgba(31, 27, 24, 0.3)` に薄める
- タイトル `<input>` に `.note-title-input` を適用：
  - `font-size: 24px`
  - `line-height: 1.6`
  - `letter-spacing: 0.06em`
- 保存状態ラベルの「余韻を保存しました」に `.save-afterglow` を適用。
  墨が和紙に滲んで消えるように、`zanshin-fade` で 2.4 秒かけて
  ふわっと現れ、ゆっくり退いていく。

### 紙の質感

- 既存の和紙テクスチャ（SVG turbulence）と差し色（金・藍）はそのまま維持。
- エディタ側に `.editor-paper` を追加し、画面全体の紙の上に
  さらに「もう一枚薄い和紙」が乗っているような層を作った。
- 質感はあくまで「気配」。文字の読みやすさを落とさない強度に保っている。

### メモカード改善

- ラベル `最後の余韻` / `残した日` のサイズを `11px → 10px`、
  letter-spacing を `0.14em → 0.12em`、不透明度を `0.7 → 0.55` に下げて
  ラベルが主役にならないよう抑えた。
- 本文プレビューの `line-height` を `golden(1.618)` から `ample(1.85)` に変更。
  2 行プレビューでも息ができるように。
- カード内の要素は変更せず、`タイトル / 最後の余韻 / 残した日 / 金のしるし` の
  最小構成は維持。

### FAB 調整

- 黒一色 (`bg-sumi`) から、墨色 `rgba(31, 27, 24, 0.92)` に変更。
- ホバー時の藍色化 (`hover:bg-indigo`) を削除し、色は静かに保つ。
- 影を `shadow-paper-hover` から、より柔らかい `0 13px 34px rgba(31, 27, 24, 0.18)`
  に差し替え。
- `+` アイコンのストロークを `1.5 → 1.3` に細くし、サイズも `22 → 20` に。
- 55px / 黄金比スケールのサイズは維持。

### 英語サブコピー全体

- 新しい `.english-subcopy` クラスを定義：
  - `font-size: 12px`
  - `line-height: 1.8`
  - `letter-spacing: 0.12em`（旧 0.18〜0.42em より弱め）
  - `color: rgba(31, 27, 24, 0.42)`
- ホームの `ZANSHIN NOTES`、空状態の英文、検索結果なしの英文、
  保存ステータス英文をすべて `.english-subcopy` に揃え、密度を一段下げた。

## 使用したフォント方針

- **display**:
  `"Hiragino Mincho ProN", "Yu Mincho", "Shippori Mincho", "Noto Serif JP", "Noto Serif", serif`
- **writing**:
  `"Hiragino Mincho ProN", "Yu Mincho", "Shippori Mincho", "Noto Serif JP", "Noto Serif", serif`
- **ui**:
  `"Hiragino Sans", "Noto Sans JP", system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`

外部 Web フォントは読み込まない。
将来 `Klee One` などの手書き調を導入する余地は CSS 変数として残してある
（その場合は `font-display: swap` 必須、ウェイトは 1〜2 個まで）。

## 残心らしさの確認

### 説明より体験で伝わるか？

OK — 残心の説明文はノート 0 件のときの導入だけになり、
通常画面では「書いたあとにも、心が残る。」の一文だけが静かに置かれる。

### 書く画面に余韻があるか？

OK — エディタ本体が `.editor-paper` で和紙として独立し、
本文の行間 2.05、letter-spacing 0.03em で字に呼吸が生まれた。
保存表示は `.save-afterglow` で滲むように現れて消える。

### 文字量は減ったか？

OK — ホームカードの英語タグラインを削除、
ノートがあるときは残心の定義文を非表示、
エディタの英語サブタイトルを削除、
空状態の本文も短く差し替えた。

### フォントは美しく読みやすいか？

OK — 見出し / 書字 / UI を CSS 変数で明確に分離。
iOS / macOS では Hiragino Mincho / Hiragino Sans が選ばれ、
他環境では Yu Mincho / Noto Serif JP / Noto Sans JP にフォールバック。

## 残っている課題

- 本文フォントを `Klee One` 等の手書き調にすると、より「紙に置く」感覚が出る。
  ただし Web フォント読み込みコストと App 化の見通しを踏まえて、Phase 10 以降で検討。
- 保存後の余韻表現は文字フェードのみ。本文エリア端の淡い墨にじみ（円相）も
  仕様には書かれているが、派手になりやすいため今回は控えた。
- メモカードの本文プレビューは「最後の行」を出す方針を維持しているが、
  「最初の行」と「最後の行」で印象が変わるかは今後 UX 検証したい。
