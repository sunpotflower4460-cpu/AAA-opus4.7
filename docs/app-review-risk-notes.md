# App Review Risk Notes

## 1. mock Premiumを本番に残す
リスク:
- 購入処理が本物でない場合、審査で問題になりやすい

対策:
- 本番提出前に StoreKit へ差し替える

## 2. 広告SDKのPrivacy申告漏れ
リスク:
- 第三者 SDK のデータ収集申告漏れ

対策:
- SDK ドキュメントと Privacy Manifest を確認し、App Privacy に反映する

## 3. ATT未対応
リスク:
- トラッキング広告や IDFA 利用時に ATT 未対応

対策:
- 非パーソナライズド広告を優先する
- 必要なら ATT を実装する

## 4. 誇大表現
リスク:
- 心理的効果や治療効果を示唆すると問題化しやすい

対策:
- メモ / 日記 / 記録アプリとして説明する

## 5. WebViewだけで低品質に見える
リスク:
- App Store で品質不足に見える可能性がある

対策:
- iOSらしい操作性
- オフライン保存
- 安定動作
- アイコン / スクリーンショット / UX品質を整える
